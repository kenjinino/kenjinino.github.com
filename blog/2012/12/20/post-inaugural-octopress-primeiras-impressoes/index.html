Post Inaugural: Octopress, Primeiras Impressões

Dec 20th, 2012 | Comments

Este é o primeiro post do meu blog pessoal.
Utilizei o Octopress como plataforma para o blog, baseado no Jekyll utilizado pelo Github Pages. Não cheguei a testar o Wordpress e o máximo que usei na vida foi o Blogger muitos anos atrás, então é praticamente a primeira vez que faço uso de uma ferramenta dessa categoria. Uma das coisas que me deixou intrigado foi a descrição do Octopress em seu próprio site:

Octopress: A blogging framework for hackers.

Ao ler a documentação da ferramenta, o motivo do título foi ficando mais claro. Já no início ele traz o aviso que se você não tem conhecimentos de git e não está habituado a usar a linha de comando, provavelmente o Octopress não seja o ideal.

Pelo fato dele estar desenvolvido em ruby (foi o que me chamou a atenção, por ser minha linguagem favorita), muitas das ferramentas utilizadas internamente fazem parte do cotidiano de um rails developer, como o Bundler (gerenciador de dependências) e Rake (automatiza tarefas definidas em um arquivo, assim como no make), entre outros.

O Octopress ao contrário do Jekyll tem um setup time menor, pois no último é necessário escrever seus próprios templates html, css e javascripts, enquanto no primeiro o usuário pode utilizar um tema padrão ou escolher alguma outra já pronta (o que é um alívio para desenvolvedores já que normalmente criar designs bonitos para páginas não é exatamente um dom natural para nós).

Um outro ponto interessante nessa plataforma é a simplicidade no deployment. Podemos realizar essa tarefa para três destinos diferentes, com links para suas instruções:

  • Github Pages;
  • Heroku;
  • Servidor particular utilizando Rsync.

Os dois primeiros são de graça e escolhi o Github Pages por já possuir uma conta no próprio Github. E seguindo os procedimentos de deploy, com algumas rake tasks o blog já estava no ar com a url padrão http://username.github.com que pode ser customizada posteriomente.

Ao contrário do que se espera de sistemas webs, o conteúdo do blog não fica armazenado em bancos de dados, ao invés disso os posts (escritos em Markdown, uma markup language) se situam em arquivos numa pasta do código fonte, sendo assim versionados pelo git, podendo ser redigidos pelo usuário no editor de texto de sua escolha.

É possível colocar o blog no ar a partir do zero em poucas horas e customizar o que lhe for conveniente em seu layout. Passado algumas horas entendendo como esta plataforma de blog funciona, dá para se dizer que o Octopress está mais para um blogging framework para developers.
